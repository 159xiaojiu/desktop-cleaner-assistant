# 桌面清理小助手 — 产品需求与实现规格（SPEC）

> 一句话定义：一个面向小白的 Windows 桌面端「整合面板」小工具，帮你把乱七八糟的桌面图标**自动分类整理**，并能**强力卸载那些卸不掉、卸了还在**的顽固软件。

- 文档版本：v1.0
- 适用系统：Windows 10 / 11（64 位）
- 阅读对象：你自己（电脑/编程小白）+ 帮你写代码的 AI
- 维护说明：每完成一个阶段，回到对应章节打勾，记录实测数字。

## 1. 项目目标

| 编号 | 目标 | 大白话解释 |
| --- | --- | --- |
| G1 | 桌面图标自动分类整理 | 把桌面上一堆图标按"浏览器/办公/游戏…"归到几个文件夹里，桌面瞬间清爽 |
| G2 | 可预览、可一键还原 | 整理前先给你看"我打算怎么放"，整理后不满意能一键变回原样 |
| G3 | 强力卸载顽固软件 | 那些在"控制面板"里卸不掉、或卸完还残留的软件，能彻底清掉 |
| G4 | 小白也能用 | 全程点鼠标，不用敲命令；最终是一个双击就能开的 exe |

非目标（这版**不做**，避免越做越大）：

- 不做垃圾文件深度清理（C 盘瘦身、缓存清理）——那是另一个大工程。
- 不做开机自启管理、内存加速、广告拦截等 360 全家桶功能。
- 不做"像 Fences 那样把图标固定摆在桌面某个框里"——只做"归类到文件夹"。

## 2. 目标用户与使用场景

- **用户画像**：电脑小白，桌面常年几十上百个图标，找东西靠肉眼扫；偶尔装了流氓软件卸不掉。
- **典型场景 1（整理）**：下班前桌面乱成一团 → 打开工具 → 点"扫描" → 看一眼预览 → 点"一键整理" → 桌面只剩几个分类文件夹。
- **典型场景 2（卸载）**：某软件在设置里点卸载没反应 / 卸完还在开机启动 → 打开工具 → 找到它 → 点"强力卸载" → 连残留一起清掉。
- **运行环境**：个人 Windows 电脑，单机使用，无需联网（联网仅用于首次下载卸载组件）。

## 4. 功能范围（MVP 必做 vs 以后再说）

### 4.1 MVP（第一版必须有）

桌面整理：

- F1 扫描桌面所有图标/文件（含用户桌面 + 公共桌面）。
- F2 按规则把每个图标判定到一个类别（关键词 → 扩展名 → 兜底"其他"）。
- F3 整理前**预览**："X 个图标 → 哪些类别"，不动文件。
- F4 一键整理：在桌面建分类文件夹，把图标移进去。
- F5 一键撤销：把刚才移动的全部还原回原位。

软件卸载：

- F6 列出本机已安装软件（名称、发布者、占用大小、卸载命令）。
- F7 普通卸载：调用软件自带的卸载程序。
- F8 强力卸载：拉起集成的 BCU，对卸不掉/有残留的软件做强制清理。

界面与打包：

- F9 一个图形界面面板（左侧导航：桌面整理 / 软件卸载）。
- F10 打包成单个 exe，双击即用，必要时提示"用管理员身份运行"。

### 4.2 以后再说（明确不在第一版）

- 自定义分类规则编辑界面（先用内置规则）。
- 定时/自动整理、监控新图标自动归类。
- 垃圾文件清理、启动项管理、浏览器插件清理。
- 多语言、皮肤主题切换。

## 5. 能力边界与风险说明（小白必读）

这一章回答你最关心的："电脑/这个工具到底能干啥、不能干啥、会不会搞坏？"

### 5.1 关于"整理桌面"

- 桌面上的图标其实就是一些**文件**（快捷方式是 `.lnk` 文件）。"整理"= 把这些文件移动到桌面上的分类文件夹里，**不会删除、不会改内容**，所以很安全。
- 有几个图标**不是文件**，是系统特殊项：`此电脑`、`回收站`、`网络`、`控制面板`、用户文件夹。工具会**自动跳过**它们，不去动（也动不了，硬动会出问题）。
- 我们**不采用**"用接口把图标摆到桌面某个坐标"的做法（像某些工具那样），因为它在重启、换屏幕分辨率、资源管理器崩溃后容易乱掉。我们用"移动到文件夹"，重启也不会变。
- 兜底保障：每次整理都会留一份"移动记录"，可一键还原（见第 9 章）。

### 5.2 关于"卸载软件"——重点说边界

先把真相讲清楚，避免你有不切实际的期待：

- **能做到**：把正常安装的软件（在"应用和功能"里能看到的）调用其卸载程序卸掉；用 BCU 强制卸载那些卸载程序坏了的软件，并清理它残留的文件夹、注册表项、开机启动项。
- **不一定能做到 / 有风险的**：
  - 真正的**病毒、木马、带 rootkit 的流氓软件**——它们会反复自我恢复，这类应交给杀毒软件（Windows Defender / 火绒等）专门处理，本工具不保证清除。
  - **系统自带组件、和系统深度绑定的东西**——强删可能让系统不稳定，工具会标记为"受保护/不建议卸载"。
  - 有些软件装了**驱动级/服务级**组件，需要重启后才能彻底清干净。
- **权限**：卸载、清残留通常需要**管理员权限**。工具启动时若权限不够，会提示你"右键 → 以管理员身份运行"。
- **为什么集成 BCU 而不是自己写**：彻底卸载涉及注册表、服务、计划任务等很多坑，BCU（1.9 万 star、开源、被广泛验证）比我们临时写的可靠得多，且对系统保护项有内置白名单，更安全。

### 5.3 误删 / 误整理怎么恢复

| 情况 | 恢复办法 |
| --- | --- |
| 桌面整理后想变回去 | 点工具里的"撤销上次整理"，按移动记录还原 |
| 文件夹被移动后找不到 | 移动记录 json 里有"原路径→新路径"，可手动找回 |
| 卸载误删了想要的软件 | 软件卸载不可逆，重新下载安装；所以卸载前工具会二次确认 |
| 工具崩溃/电脑重启 | 移动记录保存在硬盘，重开工具仍可读取并还原 |

### 5.4 这个工具的明确边界（一句话总结）

> 它是"桌面归类器 + 卸载遥控器"，不是杀毒软件，不是系统优化大师。安全第一：能预览、能撤销、危险操作有二次确认、系统关键项一律不碰。

## 6. 总体成功标准（怎么算"做成了"）

整个产品满足以下全部，才算第一版成功：

- S1 **真能用**：在一台真实有几十个图标的 Windows 电脑上，点几下就完成桌面整理，桌面明显变清爽。
- S2 **不出事**：整理过程零误删、零文件损坏；系统特殊项（此电脑/回收站等）从不被动。
- S3 **可后悔**：任何一次整理都能一键还原回原样。
- S4 **能卸顽固**：对至少一个"控制面板里卸不掉/有残留"的样例软件，用强力卸载能清干净。
- S5 **小白可独立完成**：一个没看过代码的人，照着说明双击 exe 就能完成整理和卸载，全程不碰命令行。

## 8. 分阶段实现路线（按节奏走，每步可独立验收）

设计原则：**每个阶段都能单独跑、单独验收、即使后面不做了，前面也有用**。先做最安全的（只读预览），再做会动文件的，最后才碰卸载。

```mermaid
flowchart LR
    P0[阶段0 环境准备] --> P1[阶段1 扫描+预览 只读]
    P1 --> P2[阶段2 一键整理+撤销]
    P2 --> P3[阶段3 卸载整合]
    P3 --> P4[阶段4 GUI面板整合]
    P4 --> P5[阶段5 打包+体验优化]
```

技术栈固定为：**Python 3.10+**、界面用 **CustomTkinter**、打包用 **PyInstaller**、卸载集成 **Bulk Crap Uninstaller（便携版）**。

---

### 阶段 0：环境准备（地基）

- 目标：把开发/运行环境装好，能跑通一个最小程序。
- 要做：
  1. 安装 Python 3.10+（安装时勾选"Add to PATH"）。
  2. 建项目文件夹，装依赖：`customtkinter`、`pywin32`、`pyinstaller`。
  3. 写一个能弹出空窗口的 `hello.py` 跑通。
- 验证指标（达标才算过）：
  - 命令行 `python --version` 显示 3.10 以上。
  - 运行 `hello.py` 能弹出一个窗口、能关闭，无报错。
- 完成标准：环境零报错，最小窗口能开能关。

---

### 阶段 1：扫描 + 分类预览（只读，绝对安全）

- 目标：扫描桌面、算出每个图标"应该去哪一类"，**只打印结果，不动任何文件**。
- 要做：
  1. 读取两个桌面路径：`%USERPROFILE%\Desktop`、`%PUBLIC%\Desktop`。
  2. 过滤掉系统特殊项（此电脑/回收站/网络/控制面板等）。
  3. 对每个文件按规则判类：**关键词匹配 → 扩展名 → 兜底"其他"**。
     - 预设类别：浏览器 / 办公 / 开发 / 游戏 / 社交 / 影音 / 工具 / 文档 / 图片 / 压缩包 / 安装包 / 其他。
  4. 输出一张表："图标名 → 判定类别"。
- 验证指标：
  - **分类准确率 ≥ 90%**（拿 50 个真实图标手工核对）。
  - **改动文件数 = 0**（这一步绝不能动文件，确认桌面文件数量、时间戳没变）。
- 完成标准：能稳定输出预览表，且全程没碰过任何文件。

---

### 阶段 2：一键整理 + 一键撤销（开始动文件，但可回退）

- 目标：把预览变为现实——在桌面建分类文件夹并移动图标进去，同时生成可还原的记录。
- 要做：
  1. 整理前再次显示预览，要用户确认。
  2. 在桌面创建需要的分类文件夹（已存在则复用）。
  3. 逐个移动文件，**每移动一个就记一条**"原路径 → 新路径"到移动记录 json（见第 9 章）。
  4. 提供"撤销上次整理"：读 json，把文件逐条移回原位。
  5. 处理重名冲突（同名文件移入时自动加后缀，如 `名字(1)`）。
- 验证指标：
  - 整理后桌面**只剩**分类文件夹 + 系统项。
  - **撤销成功率 = 100%**（撤销后文件数量、位置与整理前完全一致）。
  - **误删率 = 0**（整理前后文件总数不变）。
  - 整理耗时 ≤ 10 秒 / 100 个图标。
- 完成标准：能整理、能 100% 还原、无丢文件。

---

### 阶段 3：卸载整合（站在 BCU 肩膀上）

- 目标：在工具里能看到已装软件并卸载；顽固的交给 BCU。
- 要做：
  1. 读取已安装软件列表（来自 Windows 注册表卸载项：
     `HKLM\...\Uninstall`、`HKCU\...\Uninstall`，含 32/64 位），展示名称/发布者/大小。
  2. "普通卸载"：调用该软件登记的卸载命令（`UninstallString`）。
  3. "强力卸载"：检测是否已带 BCU 便携版；没有则引导下载/放入指定目录，然后拉起 BCU 对选中软件做强制卸载 + 清残留。
  4. 卸载前**二次确认**弹窗（不可逆操作）。
  5. 系统保护项标记为"不建议卸载"，默认禁用按钮。
- 验证指标：
  - **软件识别率 ≥ 95%**（对照"应用和功能"清单核对）。
  - 普通卸载能成功拉起至少 3 个软件的卸载流程。
  - 对 1 个样例顽固软件，**强力卸载后残留清零**（安装目录、注册表项搜不到）。
- 完成标准：列表准、普通卸载通、强力卸载能清残留、危险操作有确认。

---

### 阶段 4：GUI 面板整合（合成一个工具）

- 目标：把"桌面整理"和"软件卸载"合进一个图形界面，纯点鼠标完成。
- 要做：
  1. CustomTkinter 主窗口，左侧导航两项：**桌面整理 / 软件卸载**。
  2. 桌面整理页：扫描 → 预览表 → 一键整理 / 撤销 按钮 + 进度提示。
  3. 软件卸载页：软件列表 + 搜索框 + 普通卸载 / 强力卸载 按钮。
  4. 统一的确认弹窗、结果提示、错误提示（如"无图标""未找到 BCU"）。
- 验证指标：
  - **不碰命令行**就能完成整理和卸载两件事。
  - 关键操作都有可见反馈（进度/成功/失败）。
  - 异常输入不崩溃（空桌面、无软件列表、无 BCU 等）。
- 完成标准：一个界面跑通全部 MVP 功能。

---

### 阶段 5：打包 + 体验优化（交付）

- 目标：打成 exe，干净电脑双击即用。
- 要做：
  1. PyInstaller 打包成单文件 exe（带图标）。
  2. 需要管理员权限时提示/请求提权（卸载、清残留场景）。
  3. 完善异常处理与提示文案；写一页《使用说明》。
  4. 在一台尽量"干净"的 Windows 上实测。
- 验证指标：
  - **双击 exe，启动到可点击 ≤ 5 秒**。
  - 在没装 Python 的电脑也能运行。
  - **小白可用性**：找 1 个不懂电脑的人，仅凭 exe + 一页说明独立跑通整理 + 卸载。
- 完成标准：exe 可分发、可独立运行、小白能用。

## 9. 数据与备份设计（"能撤销"靠它）

"能一键还原"的秘密就是：每次整理都把"谁从哪搬到哪"记下来。

### 9.1 移动记录文件

- 存放位置：工具数据目录（如 `%APPDATA%\桌面清理助手\history\`），**不放在桌面**，避免被自己整理掉。
- 每次整理生成一个带时间戳的 json，例如 `move_20260609_1306.json`。
- 结构示意：

```json
{
  "task_id": "move_20260609_1306",
  "time": "2026-06-09 13:06:00",
  "desktop_roots": ["C:/Users/23986/Desktop", "C:/Users/Public/Desktop"],
  "moves": [
    {
      "name": "Chrome.lnk",
      "category": "浏览器",
      "from": "C:/Users/23986/Desktop/Chrome.lnk",
      "to": "C:/Users/23986/Desktop/浏览器/Chrome.lnk",
      "renamed": false
    }
  ],
  "status": "done"
}
```

### 9.2 撤销逻辑

- 读取最近一次（或指定一次）记录，**按 `moves` 逆序**把每个文件从 `to` 移回 `from`。
- 若 `from` 处已存在同名文件（极少见），改为还原成 `名字(1)` 并在日志里标注，不覆盖。
- 还原完成后把该记录标记为 `status: reverted`，保留一段时间（如 30 天）后自动清理旧记录。

### 9.3 卸载侧的数据

- 卸载本身不可逆，不做"撤销卸载"。
- 但每次卸载操作写一条**操作日志**（软件名、时间、方式：普通/强力、结果），方便事后排查。

## 10. 风险与回滚预案

| 风险 | 可能后果 | 预防 / 回滚 |
| --- | --- | --- |
| 整理时移动出错（如磁盘满、权限不足） | 部分文件移了、部分没移 | 边移边记录；出错即停并提示，已移动的可用撤销回滚 |
| 同名文件冲突 | 覆盖丢文件 | 永不覆盖，自动加后缀 `名字(1)` |
| 误把系统项当文件处理 | 桌面异常 | 内置系统项白名单，扫描阶段就排除 |
| 撤销时记录文件损坏/丢失 | 无法自动还原 | 记录存在 APPDATA 非桌面；json 损坏时给出"原路径→新路径"明细供手动还原 |
| 卸载误删需要的软件 | 软件没了 | 不可逆操作前二次确认弹窗，列出将影响项 |
| 强力卸载碰到系统/驱动组件 | 系统不稳定 | 系统保护项默认禁用卸载；提示重启后再清 |
| 权限不足导致卸载/清残留失败 | 操作无效 | 检测权限，提示"以管理员身份运行" |
| BCU 未就位 | 强力卸载点了没反应 | 启动时检测，缺失则引导下载/放入指定目录，并给出明确提示 |
| 打包后的 exe 被杀毒误报 | 用户不敢用 | 说明文档解释（PyInstaller 常见误报）；保留源码可自查 |
